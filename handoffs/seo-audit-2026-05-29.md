## Executive diagnosis

The 2026-05-29 export confirms the same class of issue as 2026-05-17, now with more evidence: index coverage and impressions continue growing, but high-intent calculator/trade-up queries still rank poorly and key blog pages have severe CTR dilution from URL/canonical variants.

Highest-leverage fix remains making every public static/prerendered SEO route expose route-specific first-class HTML metadata, then enforcing that in build/deploy verification so nginx/static routing cannot silently serve homepage metadata again.

## Code changes made from this audit

1. Centralized static SEO route definitions into `server/static-seo-pages.ts` so Express crawler handlers and prerender verification share the same expected metadata.
2. Added `scripts/seo-html.ts` with expected route metadata, route normalization, prerender head normalization, and verification helpers.
3. Updated `scripts/prerender.ts` to prerender every current blog post from `blogPosts` instead of a stale hard-coded list.
4. Updated prerender output to rewrite title/description/canonical to route-specific expected values after `dedupeHead`.
5. Added `scripts/verify-seo-html.ts` and wired it into `postbuild` so generated `dist/**/*.html` must contain exactly one title, one description, and one canonical matching the expected route.
6. Added `scripts/verify-public-seo.ts` and a deploy workflow Googlebot smoke step for `/calculator`, `/faq`, `/features`, and `/blog/how-cs2-trade-ups-work/`.
7. Changed internal blog links to canonical trailing-slash URLs from blog index, related posts, FAQ links, and homepage blog cards.
8. Marked Steam auth links `rel="nofollow"` to reduce crawler/auth noise.
9. Added regression tests covering SEO HTML verification, canonical blog links, and auth nofollow behavior.
